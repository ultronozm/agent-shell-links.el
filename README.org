#+title: agent-shell-links: Bookmarks and Org links for agent-shell sessions

Small add-on for [[https://github.com/xenodium/agent-shell][agent-shell]] that stores stable pointers to agent sessions.

An =agent-shell= session pointer records the session id, the agent identifier, and the working directory.  Opening the pointer first reuses a live matching =agent-shell= buffer when one exists; otherwise it resumes the session through =agent-shell=.

Loading this package only defines functions.  It does not change bookmark behavior or register an Org link type.  To use those features, you must call the setup functions from your init file.  See the Usage section below.

* Usage

** Emacs bookmarks

#+begin_src emacs-lisp
(require 'agent-shell-links)
(agent-shell-links-bookmark-setup)
#+end_src

Then run =M-x bookmark-set= from an =agent-shell= buffer to store a bookmark for the current session.

Jump back with =M-x bookmark-jump=.

** Org links

#+begin_src emacs-lisp
(require 'agent-shell-links)

(with-eval-after-load 'ol
  (org-link-set-parameters
   "agent-shell"
   :follow #'agent-shell-links-org-follow
   :store #'agent-shell-links-org-store))
#+end_src

Then run =M-x org-store-link= from an =agent-shell= buffer and insert it with =M-x org-insert-link=.

Stored links look like:

#+begin_example
[[agent-shell:SESSION-ID?agent=codex&dir=/path/to/project][Session title]]
#+end_example

* Example configuration

Here is a complete example config using [[https://github.com/progfolio/elpaca][elpaca]]:

#+begin_src emacs-lisp
(use-package agent-shell-links
  :ensure (:host github :repo "ultronozm/agent-shell-links.el")
  :after agent-shell
  :demand
  :config
  (agent-shell-links-bookmark-setup)
  (with-eval-after-load 'ol
    (org-link-set-parameters
     "agent-shell"
     :follow #'agent-shell-links-org-follow
     :store #'agent-shell-links-org-store)))
#+end_src
